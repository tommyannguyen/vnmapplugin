# VnMapPlugin — Full Documentation Refactor & Cleanup

**Date:** 2026-05-29
**Status:** Approved

---

## Overview

Complete refactor and cleanup of all documentation and user-facing source files in the `vnmapplugin` repository. The project is a React Native Maps SDK forked from `@rnmapbox/maps`. Most files still carry old branding, package names, and references from the upstream fork. This spec covers a systematic search-replace pass followed by targeted manual edits to make every user-facing file consistently reflect the VnMapPlugin brand.

---

## Scope

### Files Modified

- `docs/*.md` (~50 files)
- `README.md`, `CONTRIBUTING.md`, `RELEASE.md`, `LICENSE.md`
- `ios/install.md`, `android/install.md`, `android/src/main/java/README.md`
- `plugin/install.md` (if present)
- `src/vmapPlugin.native.ts` — `StyleURL` enum values
- Doc generation templates in `scripts/` that produce `import { X } from '@rnmapbox/maps'`

### Files Deleted

- `CHANGELOG.md` — references external rnmapbox release log; irrelevant to VnMapPlugin
- `README-v10.md` — legacy rnmapbox v10 setup guide; no longer applicable

### Files NOT Touched

- `__tests__/` — functional behavior unchanged
- Internal Java/Kotlin package names (`com.rnmapbox.rnmbx`) — native implementation details, not user-facing
- `node_modules/`, `lib/`, `.git/`, lock files

## Execution Phases

### Phase 1 — Deletions
- Delete `CHANGELOG.md`

### Phase 2 — Global Search-Replace Pass
Run in this order to avoid double-replace issues:
1. `@vnmapplugin/maps-example` → `@vnmapplugin/maps-example`
2. `@rnmapbox/maps` → `@vnmapplugin/maps`
3. `rnmapbox/maps` → `tommyannguyen/vnmapplugin`
4. `RNMapboxMaps` → `RNVnMapMaps` (covers all suffixed variants)
5. `MapboxGL` → `VnMapPlugin`
6. `Mapbox` → `VnMapPlugin` (after MapboxGL to avoid partial matches)

Exclude: `.git/`, `node_modules/`, `lib/`, `yarn.lock`, `*.lock`

- Handle case-by-case: some may be in live tile server URLs (leave as-is until VnMapPlugin URLs confirmed) or in code comments that are contextually accurate
